# Backend Feature Specs

Every backend feature needs a spec before implementation.

Use `_FEATURE_SPEC_TEMPLATE.md` for work involving APIs, database changes, RLS, auth, Stripe, jobs, notifications, vendors, audit logs, registration conversion, or child-data permissions.

For cross-repo work, the backend spec is the lead spec when it changes data contracts or privileged behavior. Link web/mobile/site specs when UI work has separate implementation steps.
